# 🛠️ 段階的リファクタリング計画（3-1）

> 目的: 現行構造から本テンプレートの4層へ安全に移行するための段階計画を定義する。

## 計画の作り方
1. 構造マッピング
   - [ ] 現行の UI/状態/データアクセス/ビジネスロジックの所在を一覧化
   - [ ] 目標構造（Core と Features 4層）への対応表を作成
2. パイロット選定
   - [ ] 影響が少なく価値の高いフィーチャーを1つ選び、先に移行
3. 互換アダプタ方針
   - [ ] 旧コードを暫定的にラップするアダプタ／ファサードを定義（段階移行を可能にする）
4. マイルストーン
   - [ ] M1: Core の最小骨格作成（routing/theme/api/exceptions/database）
   - [ ] M2: 先行フィーチャーの4層化完了
   - [ ] M3: 残存フィーチャーの段階移行計画合意

## 成功の指針
- 小さく切って安全に移行（Big Bang禁止）
- 互換アダプタで旧/新の接続を管理し、段階的に置換
