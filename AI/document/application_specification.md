# シンプルメモアプリ - アプリケーション仕様書

## 📋 基本情報

- **作成日**: 2025年9月1日
- **アプリ名**: シンプルメモ
- **バージョン**: 1.0.0
- **プラットフォーム**: Flutter (iOS/Android)

## 🎯 アプリケーション概要

### コンセプト
余計な機能を省いた、使いやすさ重視のシンプルなメモアプリ

### 解決する課題
- 複雑な機能に惑わされず、素早くメモを取りたい
- シンプルな操作でストレスなくメモ管理をしたい
- 必要最小限の機能で十分なメモアプリが欲しい

## 👥 ターゲットユーザー

- **主要ターゲット**: 日常的にメモを取る全ての人
- **利用シーン**: 
  - 思いついたアイデアの記録
  - 買い物リストの作成
  - 簡単なタスク管理
  - 会議や授業での簡単なメモ
- **ユーザー特性**: シンプルな操作を好み、素早いメモ取りを重視する

## 🔧 機能仕様

### 必須機能

#### 1. メモ作成機能
- **概要**: 新しいメモを作成する
- **入力項目**:
  - タイトル（必須、最大50文字）
  - 本文（任意、最大1000文字）
- **操作**: FABボタンまたはヘッダーボタンから作成画面へ遷移

#### 2. メモ一覧表示機能
- **概要**: 作成したメモの一覧を表示する
- **表示項目**:
  - タイトル
  - 本文の冒頭（最大2行まで）
  - 作成日時
- **並び順**: 作成日時の降順（新しいものが上）
- **操作**: タップで編集画面へ遷移

#### 3. メモ編集機能
- **概要**: 既存のメモを編集する
- **操作**: 一覧からメモをタップして編集画面へ遷移
- **編集項目**: タイトル、本文
- **保存**: 自動保存または明示的な保存ボタン

#### 4. メモ削除機能
- **概要**: 不要なメモを削除する
- **操作**: 
  - 編集画面からの削除ボタン
  - 一覧画面でのスワイプ削除
- **確認**: 削除前に確認ダイアログを表示

#### 5. メモ検索機能
- **概要**: タイトルや本文でメモを検索する
- **検索対象**: タイトル、本文
- **検索方式**: 部分一致検索
- **操作**: ヘッダーの検索アイコンから検索バーを表示

## 📱 画面設計

### 1. メモ一覧画面（ホーム画面）
- **レイアウト**: 
  - ヘッダー（タイトル + 検索アイコン）
  - メモリスト（スクロール可能）
  - FAB（新規作成ボタン）
- **操作**:
  - メモタップ → 編集画面へ遷移
  - FABタップ → 新規作成画面へ遷移
  - スワイプ → 削除確認

### 2. メモ作成・編集画面
- **レイアウト**:
  - ヘッダー（戻るボタン + 保存ボタン + 削除ボタン※編集時のみ）
  - タイトル入力フィールド
  - 本文入力フィールド（複数行）
- **操作**:
  - 戻るボタン → 一覧画面へ戻る
  - 保存ボタン → メモを保存して一覧画面へ戻る
  - 削除ボタン → 削除確認ダイアログ表示

### 3. 検索画面
- **レイアウト**:
  - 検索バー
  - 検索結果リスト
- **操作**:
  - 検索結果タップ → 編集画面へ遷移

## 💾 データ仕様

### メモエンティティ
```dart
class Note {
  final String id;          // 一意識別子（UUID）
  final String title;       // タイトル（最大50文字）
  final String content;     // 本文（最大1000文字）
  final DateTime createdAt; // 作成日時
  final DateTime updatedAt; // 更新日時
}
```

### データ保存
- **保存場所**: ローカル端末（SQLite）
- **同期**: なし（ローカルのみ）
- **バックアップ**: 端末のバックアップ機能に依存

## 🎨 UI/UX仕様

### デザインガイドライン
- **デザインシステム**: Material Design 3
- **カラーテーマ**: システムのライト/ダークモードに対応
- **フォント**: システムデフォルトフォント
- **アニメーション**: 最小限（画面遷移のみ）

### ユーザビリティ
- **操作性**: 片手操作を考慮したレイアウト
- **アクセシビリティ**: 基本的なアクセシビリティ対応
- **レスポンシブ**: 異なる画面サイズへの対応

## 🔧 技術仕様

### 開発環境
- **フレームワーク**: Flutter
- **状態管理**: Riverpod
- **データベース**: SQLite (sqflite)
- **ルーティング**: GoRouter
- **UI補助**: Flutter Hooks

### アーキテクチャ
- **パターン**: クリーンアーキテクチャ
- **レイヤー構成**:
  - Domain層: エンティティ、リポジトリインターフェース、ユースケース
  - Infrastructure層: データソース、リポジトリ実装
  - Application層: 状態管理、プロバイダー
  - Presentation層: UI、ページ、ウィジェット
